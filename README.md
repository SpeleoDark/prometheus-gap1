## GAP-4: Визуализация метрик в Grafana

### Цель
Настроить Grafana для визуализации метрик, собранных Prometheus, и создать дашборды для мониторинга инфраструктуры и CMS.

### Выполненные шаги

1. **Установка Grafana**  
   - Добавлен официальный репозиторий, установлен пакет `grafana`.  
   - Сервис запущен и добавлен в автозагрузку.

2. **Настройка источника данных**  
   - Добавлен источник данных Prometheus (`http://localhost:9090`).  
   - Проверено успешное подключение.

3. **Создание папок**  
   - Созданы папки: `infra` (для инфраструктурных дашбордов) и `app` (для дашбордов CMS).

4. **Дашборд инфраструктуры (`infra`)**  
   - **Infrastructure Overview** включает панели:  
     - CPU Usage  
     - Memory Usage  
     - Disk Usage (root)  
     - Network Traffic (in/out)  
     - System Load (1, 5, 15)  
     - System Uptime

5. **Дашборд CMS (`app`)**  
   - **CMS Health** включает панели:  
     - Доступность компонентов (nginx, php-fpm, postgresql, blackbox)  
     - Время ответа главной страницы  
     - Активные соединения nginx  
     - Количество запросов в секунду (nginx)  
     - Активные процессы php-fpm  
     - Количество подключений к PostgreSQL

6. **Задание со звёздочкой**  
   - Настроен алерт в Grafana о недоступности Nginx с уведомлением через Telegram.  
   - Создан drill-down дашборд с переменной `instance` для детального просмотра.

### Результаты
- Скриншоты дашбордов находятся в папке `GAP-2` этого репозитория.
- Описание дашбордов и использованных метрик приведено в `GAP-2/README.md`.

### Используемые метрики
- `node_*` – метрики инфраструктуры (CPU, память, диск, сеть, загрузка)  
- `up` – доступность компонентов  
- `probe_success`, `probe_duration_seconds` – данные blackbox exporter  
- `nginx_*` – метрики nginx  
- `php_fpm_*` – метрики php-fpm  
- `pg_stat_database_*` – метрики PostgreSQL
